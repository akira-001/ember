### [MILESTONE] ambient_listener 「ちょっと」誤呼びかけ修正（夜セッション）
- **対象ファイル**: `open-claude/scripts/voice_chat/ambient_listener.py` / `tests/test_ambient_listener.py`
- **事象**: 19:43:59 JST に Akira 発言ゼロで Mei が「お疲れ様。ゆっくりしててね。」を発話（ambient/tier1 gemma4:e4b 経路）
- **根本原因 3 段ハマり**:
  1. Whisper が無音区間で「お疲れ様」「ちょっと」を幻聴（無音区間 hallucination filter は単独時のみ作動）
  2. 「ちょっといけると言うかも 待ちてばっかり」+「お疲れ様でした」が同 batch に集約 → `batch low-value bypass (call=True)` 発動
  3. `_USER_CALL_RE = r'メイ|ねえ|ねぇ|ちょっと'` が「ちょっと」にマッチ → `current_speaker == None` でも `user_initiative` 確定 → `intervention=reply` で tier1 LLM 起動
- **修正内容**:
  - `_USER_CALL_RE` から「ちょっと」除去（過汎用キーワード、Whisper 幻聴源）
  - `_STRONG_USER_CALL_RE = re.compile(r'メイ')` 新設、明示的な名前呼びは声紋なしでも `user_initiative` 維持
  - 「ねえ/ねぇ」のみは声紋未識別時 `user_likely` に格下げ（`reply` ではなく長文/質問のみ `reply`、それ以外 `backchannel`）
- **テスト 3 件追加**:
  - `test_chotto_alone_does_not_promote_to_user_initiative`（事故再現）
  - `test_weak_call_without_name_downgrades_to_user_likely`
  - `test_explicit_name_call_keeps_user_initiative_without_speaker_id`
- **ライブ検証**: ambient_listener / co_view 計 61 件全通過、voice_chat 再起動完了で本番反映済

### [INSIGHT] arousal=0.7 — 声紋True前提運用は短音呼びかけを殺す
- speaker_id.identify() は ≥1.0s 要求 → Akira の「メイ」「うん」「ねぇ」は全部 `skip identify: too short` で声紋判定不能
- 「True 前提でブロック」だと会話のターンテイキングが完全死する
- 正解は **3 値非対称運用**: `True → reply` / `他人確定 → block` / `unknown → user_likely 止まり`
- 既存 `current_speaker == None` フォールバックは 2 値（True or anything-else=reply可）で誤発火のホットスポットだった

### [PATTERN] arousal=0.6 — 多層フィルタの誤発火は「どの層で素通りしたか」を順に切り分ける
- STT 層（Whisper hallucination filter）→ batch low-value 層（call=True バイパス）→ classify_source 層（USER_CALL_RE）→ decide_intervention 層 → tier1 LLM 層
- 今回は STT 層を素通り（batch内なら filter 不適用）→ batch low-value 層で `call=True` 誤判定 → classify_source で `user_initiative` 昇格、と 3 層連続で漏れた
- 修正は最も狭い責務の層（classify_source の正規表現分離）で済んだ
