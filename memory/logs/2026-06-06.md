### [INVESTIGATION] 09:30 — Claude Code CLI Opus 4.8 切替不能の原因究明
*Arousal: 0.6 | Emotion: 探究*

Akiraさんが `/model` で「Default (recommended) Opus 4.8 with 1M context」を選んでも反映されず、毎回 Opus 4.7 に戻る現象。原因究明:

1. `~/.claude.json` の `additionalModelOptionsCache: []` が空 → CLI が利用可能モデル一覧として 4.8 を認識していない
2. `opus48LaunchSeenCount: 4` は出ているのにキャッシュが更新されない CLI バグ
3. settings.json の `model: "claude-opus-4-8"` も同様に無視され Opus 4.7 にフォールバック
4. picker での「saved as your default for new sessions」も次回起動で消失

実証した workaround:
- ✅ `claude --model claude-opus-4-8` フラグ ≒ キャッシュチェックをバイパスする唯一の経路
- ❌ settings.json の `model` フィールド
- ❌ picker の「set as default」
- ❌ `opus` alias、`claude-opus-4-8[1m]` 構文も無効

→ 暫定対応として `~/.zshrc` に alias 設定済み。詳細: `project_claude_opus48_alias_workaround.md`
