### [INSIGHT] Electron の "Default - X" alias は silent stream を返す
*Arousal: 0.9 | Emotion: Frustrated→Realize*

macOS の audio device enumeration には「`Default - MacBook Pro Microphone (Built-in)`」という synthetic alias と実デバイス「`MacBook Pro Microphone (Built-in)`」が併存。Electron Chromium が前者を deviceId で pin すると 1323 byte の silent stream を返す。3 時間以上のデバッグ末に `^Default\s*[-–:]/` を score=-100 で除外する fix で解決。

**Why:** Electron の audio service が "Default" 別名を正しく解決できない既知の挙動。
**How to apply:** mic candidates を rank する時、Default- 接頭辞の deviceId は明示的に skip する。Web Chrome は同 label でも内部解決して bare device id を渡してくれるが Electron は素のまま渡る。

### [PATTERN] Health-check 連続 getUserMedia で audio service が混乱
*Arousal: 0.8 | Emotion: Realize*

`acquireHealthyMicStream` で probe stream + 候補 N 個 + default fallback と短時間に 5 つ以上の getUserMedia/AudioContext を開閉すると、macOS Continuity Mic との競合 + Electron audio service が一時的に全候補で peak=0 を返すようになる。

**修正:** ユーザーが Input セレクタで明示選択した場合は **trust-mode** に切替—health-check loop を完全 skip して 1 回だけ getUserMedia で取得。
**How to apply:** 自動選択時のみ multi-candidate health-check を使い、ユーザー指定時は probe loop 不要。Akira さんが手動で MacBook Pro Microphone (Built-in) を選んだ時に最も信頼度が高い。

### [INSIGHT] iPhone Air mic は MacBook Pro mic より RMS が低かった
*Arousal: 0.5 | Emotion: Surprise*

CDP 経由で両 mic の 5s RMS 比較を実測:

| Mic | meanRMS | maxPeak | peakRMS |
|-----|---------|---------|---------|
| iPhone Air | 0.00051 | 0.0070 | 0.0017 |
| MacBook Pro | 0.00383 | 0.0432 | 0.0098 |

MacBook Pro が約 7 倍の音量だが、いずれも RMS_THRESHOLD=0.05 には届かない（動画スピーカー → mic の物理距離が問題）。「iPhone を選べば良い」と思い込んでいたが、実際には MacBook Pro 内蔵が優位。

### [PATTERN] enrichment loop の CLI not found ループ
*Arousal: 0.7 | Emotion: Realize*

`/tmp/whisper-serve.log` に `[enrichment/claude] claude CLI not found` / `[enrichment/codex] codex CLI not found` が 60s 周期で連発。原因 2 点:
1. launchd が起動した whisper の PATH は `/usr/bin:/bin:/usr/sbin:/sbin` のみ → `claude` (`~/.local/bin`) と `codex` (`/opt/homebrew/bin`) が見つからない
2. `_last_signature` を成功時のみ更新していたため、失敗 sig は毎 60s 再試行

**修正:** `_resolve_bin()` で augmented PATH の `shutil.which` 解決 + `FAIL_COOLDOWN_SEC=600s` で同 sig は 10 分冷却 + `_last_signature` も失敗時に更新。

### [INSIGHT] launchd 経由の whisper は ~/.local/bin / /opt/homebrew/bin が PATH に無い
*Arousal: 0.7 | Emotion: Aha*

`launchctl print gui/$(id -u)/local.whisper.serve | grep PATH` で確認すると `default environment = { PATH => /usr/bin:/bin:/usr/sbin:/sbin }`。シェル起動時に追加されるパスは launchd 子プロセスには継承されない。subprocess で外部 CLI を呼ぶ Python コードは絶対パス解決必須。

**How to apply:** subprocess で `claude` / `codex` / `mlx-lm` 等を呼ぶ全コードに `shutil.which(name, path=augmented_path)` パターンを適用。`_BIN_SEARCH_PATHS` をモジュールレベルで定義しておく。
