### [INVESTIGATION] 10:15 haru — interest-scanner cron「Command failed」根本原因究明＆対策
*Arousal: 0.7 | Emotion: 探究→解決*
Eve から interest-scanner の Cron Error 通知。デジャヴチェック→手動実行は成功（exit0, 73件）→環境要因疑い。
cron-history.jsonl を全件解析した結果、**失敗30件すべてが実行時間 10〜20分**（正常13〜68秒）と判明。真因は interest_scanner.py に実行時間上限が無く、arXiv/SemanticScholar の 429 バックオフ(10/20/30s)＋15s socket timeout が13カテゴリ分累積→runtime が膨張→runner(scheduler.ts)が `execFile(python,{timeout:600s})` で起動するため SIGTERM kill→「Command failed」。コードバグでも環境バグでもなく **unbounded runtime の設計欠陥**。
対策: `RUN_BUDGET_SEC=420` の wall-clock バジェットを追加。`budget_exceeded()` でカテゴリループ途中打ち切り＋wildcard/cross スキップ→集めた分で保存・サマリ出力→exit 0。budget=0 即超過テストと通常フル実行(73件,exit0)の両経路を実データ検証。スタンドアロン python なので再起動不要、次の :50 から有効。メモリ保存済([[project_interest_scanner_timeout]])。
