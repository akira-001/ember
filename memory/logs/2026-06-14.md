
### [INVESTIGATION] scheduler-watchdog「Claude Code process aborted by user」偽アラートを根本修正
*Arousal: 0.6 | Emotion: 探究→解決*
Mei から scheduler-watchdog の Cron Error 通知。デジャヴチェック→cron-history 全期間集計で success 1094 / timeout 14 / error 32（≈4%）。決定的手がかり: 同一 startedAt で timeout と error が必ずペア記録。真因は **watchdog が決定論タスク（cron-history を1hカウントするだけ）なのに旧 message 型でフル LLM 起動**していたこと。稀に >120s ハング→scheduler.ts の timeout が abortController.abort()（507行）→SDK throw→catch（589行）が「aborted by user」記録＋Slack 偽アラート送信。「aborted by user」はユーザー中断ではなく 120s タイムアウト kill の誤表示だった。

根本修正: `packages/slack-bot/scripts/scheduler_watchdog.py`（stdlib のみ・<1s）を新規作成し cron-jobs.json を command 型に変更（timeout 30s, silentMode false）。健全時=無出力（投稿なし）、0件時のみ警告 print（scheduler が投稿）。両パスを実データ＋合成データで検証、bot 再起動でスケジューラ登録確認済み。overview.md／auto-memory／[[project_scheduler_watchdog_false_alert]] 更新。runtime>timeout 系は [[project_interest_scanner_timeout]] と同根。
