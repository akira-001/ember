⏺ スキル自動学習の完全仕様

  全体フロー

  スキル使用開始      →  [SKILL] ログ + track skill_start
      ↓
  スキル実行中        →  逸脱があれば track でイベント記録
      ↓
  スキル使用完了      →  [SKILL] ログ + track skill_end
      ↓
  タスク完了後        →  learn でメトリクス記録
      ↓
  Wrap 時            →  track-summary で改善判定
      ↓
  auto: 自動 Edit    →  SKILL.md 更新 + learn 記録
  ask: ユーザー確認   →  承認されたら Edit
  off: スキップ

  ---
  1. スキル使用の追跡（セッション中）

  使用開始（スキルの SKILL.md を読んで手順に従い始めた時）

  ログに記録:
  ### [SKILL] <skill-name> 使用開始
  *Arousal: 0.4 | Emotion: Execution*
  [タスクの概要（1行）]

  ---

  DB に記録:
  cogmem skills track "<skill-name>" --event skill_start --description "<タスクの概要>"

  逸脱イベント（手順からの逸脱が発生した瞬間にリアルタイム記録）

  cogmem skills track "<skill-name>" --event <type> --description "..." [--step "Step
  N"]

  ┌───────────────┬─────────────────────────┬──────────────────────────────────────┐
  │  event_type   │        トリガー         │                  例                  │
  ├───────────────┼─────────────────────────┼──────────────────────────────────────┤
  │ extra_step    │ スキルにない手順を追加  │ SKILL.md にない jq フィルタを追加    │
  │               │ 実行                    │                                      │
  ├───────────────┼─────────────────────────┼──────────────────────────────────────┤
  │ skipped_step  │ 手順を意図的にスキップ  │ 「Step 4 のバックアップは今回不要」  │
  ├───────────────┼─────────────────────────┼──────────────────────────────────────┤
  │ error_recover │ エラー発生→リカバリ     │ 認証エラー → ssh-agent 再起動        │
  │ y             │                         │                                      │
  ├───────────────┼─────────────────────────┼──────────────────────────────────────┤
  │ user_correcti │ ユーザーが修正指示      │ 「カレンダー名が違う」「そうじゃなく │
  │ on            │                         │ て」                                 │
  └───────────────┴─────────────────────────┴──────────────────────────────────────┘

  逸脱判定: エージェント（Claude）の自己判断。スキルの手順と実際の行動を比較して「これ
  は逸脱だ」と判断したら記録する。

  記録しないケース: スキル通りにスムーズに実行 →
  逸脱イベントなし。些末な順序変更も不要。

  使用完了

  ログに記録:
  ### [SKILL] <skill-name> 使用完了
  *Arousal: 0.4 | Emotion: Completion*
  track イベント: N件（extra_step: X, skipped_step: Y, error_recovery: Z,
  user_correction: W）
  → スムーズに実行 / 改善点あり

  ---

  DB に記録:
  cogmem skills track "<skill-name>" --event skill_end --description "<結果の概要>"

  ---
  2. メトリクス記録（タスク完了後）

  cogmem skills learn "<context>" --effectiveness 0.0-1.0 --user-satisfaction 0.0-1.0

  ベクトル検索で最も近いスキルを自動マッチし、以下を更新:
  - average_effectiveness — 指数移動平均（α=0.2）
  - total_executions — +1
  - successful_executions — effectiveness > 0.6 なら +1
  - last_used_at — 現在時刻

  DB: skill_usage_log テーブルに記録。

  ---
  3. 改善判定（Wrap 時）

  cogmem skills track-summary --date YYYY-MM-DD --json

  出力例:
  {
    "date": "2026-03-25",
    "skills_used": [
      {
        "skill_name": "morning-briefing",
        "events": [
          {"event_type": "user_correction", "description":
  "ドジャースの試合時間が間違ってた"}
        ],
        "needs_improvement": true,
        "reason": "1 user_correction(s)"
      }
    ],
    "skills_ok": ["recall", "business-strategy"]
  }

  判定ロジック（skill_start/skill_end は判定から除外）:

  ┌─────────────────────────┬──────────┐
  │          条件           │   判定   │
  ├─────────────────────────┼──────────┤
  │ user_correction 1件以上 │ 改善必須 │
  ├─────────────────────────┼──────────┤
  │ error_recovery 1件以上  │ 改善必須 │
  ├─────────────────────────┼──────────┤
  │ extra_step 2件以上      │ 改善推奨 │
  ├─────────────────────────┼──────────┤
  │ skipped_step 2件以上    │ 改善推奨 │
  ├─────────────────────────┼──────────┤
  │ 上記なし                │ 改善不要 │
  └─────────────────────────┴──────────┘

  追跡の可視性:

  ┌─────────────────────────────┬────────────────────────┐
  │          パターン           │          意味          │
  ├─────────────────────────────┼────────────────────────┤
  │ skill_start あり + 逸脱0件  │ スムーズに実行された   │
  ├─────────────────────────────┼────────────────────────┤
  │ skill_start あり + 逸脱あり │ 改善が必要             │
  ├─────────────────────────────┼────────────────────────┤
  │ skill_start なし            │ スキルが使われなかった │
  └─────────────────────────────┴────────────────────────┘

  4. 自動改善（Wrap Step 3.7）

  [cogmem.skills]
  auto_improve = "auto"  # "auto" | "ask" | "off"

  ┌───────────────────┬────────────────────────────────────────────────────────────┐
  │      モード       │                            動作                            │
  ├───────────────────┼────────────────────────────────────────────────────────────┤
  │ "auto"            │ 改善対象の SKILL.md を自動で Edit                          │
  │ (デフォルト)      │                                                            │
  ├───────────────────┼────────────────────────────────────────────────────────────┤
  │ "ask"             │ 「[スキル名] に改善点あり（理由）。更新する？」→ 承認後に  │
  │                   │ Edit                                                       │
  ├───────────────────┼────────────────────────────────────────────────────────────┤
  │ "off"             │ スキップ                                                   │
  └───────────────────┴────────────────────────────────────────────────────────────┘

  改善内容の対応表:

  ┌─────────────────┬────────────────────────────────────────┐
  │    イベント     │           SKILL.md への反映            │
  ├─────────────────┼────────────────────────────────────────┤
  │ extra_step      │ 該当箇所に手順を追加                   │
  ├─────────────────┼────────────────────────────────────────┤
  │ skipped_step    │ 条件付き実行の注記を追加 or 手順を削除 │
  ├─────────────────┼────────────────────────────────────────┤
  │ error_recovery  │ エラーハンドリング手順を追加           │
  ├─────────────────┼────────────────────────────────────────┤
  │ user_correction │ 指摘内容を反映（最優先）               │
  └─────────────────┴────────────────────────────────────────┘

  改善後に cogmem skills learn でメトリクスも記録。引き継ぎに「スキル自動改善:
  [スキル名] 更新（理由）」と記録。

  5. 健全性チェック（定期診断）

  cogmem skills audit     # Session Init (Step 5.5) で自動実行
  cogmem skills review    # 手動で全体レビュー

  ┌────────────────────────┬────────────────────────────────┬──────────────────────┐
  │        チェック        │              条件              │       priority       │
  ├────────────────────────┼────────────────────────────────┼──────────────────────┤
  │ low effectiveness      │ eff < 0.5, exec >= 3           │ high (< 0.3) /       │
  │ (used)                 │                                │ medium               │
  ├────────────────────────┼────────────────────────────────┼──────────────────────┤
  │ low effectiveness      │ eff < 0.5, exec < 3            │ low                  │
  │ (new)                  │                                │                      │
  ├────────────────────────┼────────────────────────────────┼──────────────────────┤
  │ declining trend        │ 5件以上の単調減少 + drop >=    │ high (>= 0.3) /      │
  │                        │ 0.15                           │ medium               │
  ├────────────────────────┼────────────────────────────────┼──────────────────────┤
  │ never used             │ 作成から7日以上、exec = 0      │ low                  │
  ├────────────────────────┼────────────────────────────────┼──────────────────────┤
  │ stale                  │ 60日以上未使用                 │ low                  │
  └────────────────────────┴────────────────────────────────┴──────────────────────┘

  ---
  6. 収束メカニズム

  初期: スキルの手順が不完全
    → 逸脱イベントが多い → 毎回 Wrap で SKILL.md が改善される
      → 手順が充実する → 逸脱が減る
        → track イベントが減る → 改善が発火しなくなる
          → スキルが安定状態に到達

  環境変化（ツールの仕様変更、ユーザーの好み変更等）で再び逸脱が増えれば、自動的に改善
  ループが再起動する。

  7. DB スキーマ

  -- イベントトラッキング
  CREATE TABLE skill_session_events (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      session_date TEXT NOT NULL,       -- YYYY-MM-DD
      skill_name TEXT NOT NULL,         -- スキルのディレクトリ名
      event_type TEXT NOT NULL,         --
  skill_start|skill_end|extra_step|skipped_step|error_recovery|user_correction
      description TEXT NOT NULL,
      step_ref TEXT,                    -- "Step 3" 等（任意）
      timestamp TEXT NOT NULL
  );

  -- メトリクス（既存）
  CREATE TABLE skill_usage_log (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      context TEXT NOT NULL,
      skill_id TEXT,
      effectiveness REAL,
      timestamp TEXT NOT NULL
  );
